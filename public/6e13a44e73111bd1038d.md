## はじめに

Oracle Exadata Database Service on Dedicated Infrastructure(ExaDB-D)のWhat’s Newを確認していたところ次の機能が追加されていたのでチェックしてみました。

> Improved Visibility for Autonomous AI Database on Dedicated Infrastructure

内容としては、ExaDB-DのExadata Infrastructure詳細画面で従来のExadata VM Clusterだけでなく、Autonomous AI Database on Dedicated Infrastructure(ADB-D)で利用するAutonomous Exadata VM Clusterも確認できるようになった、というOCIコンソールの表示・導線改善です。

Autonomous AI Database on Dedicated Infrastructure(ADB-D)の概要はこちら

https://speakerdeck.com/oracle4engineer/adb-d-technical-detail-jp

本記事では、公式ドキュメントの差分を確認したうえで、OCIコンソール上でどのように見え方が変わっているのかを確認します。

## 確認した公式情報

今回確認した主な公式情報は以下です。

- [What’s New in Oracle Exadata Database Service on Dedicated Infrastructure](https://docs.oracle.com/ja-jp/iaas/exadatacloud/doc/exa-whats-new.html)
- [New Feature Announcements - Autonomous AI Database on Dedicated Exadata Infrastructure](https://docs.oracle.com/ja/cloud/paas/autonomous-database/dedicated/adbaa/new-feature-announcements.html)
- [New UX Updates for Autonomous AI Database on Cloud@Customer and Dedicated Infrastructure](https://blogs.oracle.com/autonomous-ai-database/new-ux-updates-for-autonomous-ai-database-on-cloudcustomer-and-dedicated-infrastructure)

ExaDB-DのWhat’s Newでは、以下の更新として掲載されています。

- Service: Database
- Release Date: May 20, 2026
- Feature: Improved Visibility for Autonomous AI Database on Dedicated Infrastructure

公式ドキュメントでは、ExaDB-Dのランディングページ体験が改善され、ExaDB-DとAutonomous AI Databaseの両方をホストする共有Exadata Infrastructure環境の視認性が向上した、と説明されています。

具体的には、以下の2点がポイントです。

- Exadata Infrastructure詳細ページで、Exadata VM ClustersとAutonomous Exadata VM Clustersの両方が表示される
- ナビゲーションにAutonomous Exadata VM Clustersが追加され、選択するとAutonomous AI Database on Dedicated Infrastructure側のページへ遷移する

今回はOCIコンソールのUX、ナビゲーション改善がメインの内容のようです。

## OCIコンソールで確認する
せっかくなのでKIX(大阪)の環境でUIが変更されているか確認しました。
OCIコンソールにログインし、左上のナビゲーションメニューからOracle AI Database > Oracle Exadata Database Service on Dedicated Infrastructure > Exadata Infrastructureに移動します。

対象のコンパートメントを選択し、確認対象のExadata Infrastructureを開きます。
Exadata Infrastructure詳細画面を開くと、Exadata VM Clustersを確認できます。

ここで今回の更新により、Autonomous Exadata VM Clustersも同じExadata Infrastructureの画面から確認できるようになっています。以前まではExadata VMクラスタのみが表示されていました。

![image.png](https://qiita-image-store.s3.ap-northeast-1.amazonaws.com/0/3489148/7487abf7-872f-4b49-8dd2-087858717e6a.png)

### 左ナビゲーションを確認する

次に、ExaDB-Dの画面左側のナビゲーションを確認します。

今回の更新により、左ナビゲーションにAutonomous Exadata VM Clustersが追加されています。

![image.png](https://qiita-image-store.s3.ap-northeast-1.amazonaws.com/0/3489148/35d87dea-800d-4e78-b3f7-f2c2f08f755d.png)

このAutonomous Exadata VM Clustersを選択すると、Autonomous AI Database on Dedicated Infrastructure側のAutonomous Exadata VM Clustersページに遷移します。

![image.png](https://qiita-image-store.s3.ap-northeast-1.amazonaws.com/0/3489148/3366119f-b76c-4f3d-9d70-8f2a246ef022.png)

ExaDB-D側からAutonomous AI Database側へ自然に移動できるようになっているため、同一Exadata Infrastructureを共有している環境では、リソースの所在を把握しやすくなりました。

## 確認結果
今回の環境では、ExaDB-DのExadata Infrastructure詳細画面から、以下を確認できました。

- Exadata VM Clustersが表示される
- Autonomous Exadata VM Clustersが表示される
- 左ナビゲーションにAutonomous Exadata VM Clustersが表示される
- Autonomous Exadata VM Clustersを選択すると、Autonomous AI Database on Dedicated Infrastructure側の画面に遷移する

したがって、公式ドキュメントに記載されているとおり、ExaDB-DとAutonomous AI Databaseが同じExadata Infrastructureを利用している環境において、コンソール上の視認性が改善されていることを確認できました。

## まとめ
ExaDB-DのWhat’s Newに追加された `Improved Visibility for Autonomous AI Database on Dedicated Infrastructure` を確認しました。

今回のポイントは以下です。

- ExaDB-DのExadata Infrastructure詳細画面で、Exadata VM ClustersとAutonomous Exadata VM Clustersの両方を確認できるようになった
- 左ナビゲーションにAutonomous Exadata VM Clustersが追加された
- 選択するとAutonomous AI Database on Dedicated Infrastructure側のページへ遷移する
- APIやIAM、プロビジョニング動作の変更ではなく、主にUX改善

ExaDB-DとAutonomous AI Databaseを同じExadata Infrastructure上で利用している環境では、これまでよりもリソースの全体像を把握しやすくなったと思います。

特に、運用時に「このExadata Infrastructure上で、通常のExadata VM ClusterとAutonomous Exadata VM Clusterがそれぞれどう配置されているか」を確認したい場面では便利な改善です。
